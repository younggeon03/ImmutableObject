# [자바 중급] String 클래스와 문자열 처리 핵심 요약

## 1. String 클래스 기본
* 자바에서 문자열을 다루는 대표적인 **참조형 클래스**이다.
* 생성 방법은 **문자열 리터럴**(`"hello"`) 방식과 **객체 생성**(`new String("hello")`) 방식 두 가지가 있다.
* **메모리 최적화**: 자바 9 이전에는 내부적으로 `char[]`(2byte)을 사용했으나, 자바 9부터는 단순 영어/숫자의 경우 메모리 절약을 위해 `byte[]`(1byte)를 사용하도록 변경되었다.

## 2. 문자열 비교와 문자열 풀 (String Pool)
* **문자열 비교 규칙**: 객체의 주소를 비교하는 `==` (동일성 비교) 대신, 내부의 논리적 값을 비교하는 `equals()` (동등성 비교) 메서드를 항상 사용해야 한다.
* **문자열 풀 (String Pool)**:
    * 문자열 리터럴(`"..."`)로 생성하면 힙 영역의 문자열 풀에 보관된다.
    * 같은 문자가 풀에 이미 존재하면 새로운 객체를 만들지 않고 기존 참조값을 반환하여 성능과 메모리를 최적화한다.
* **new String()**: 문자열 풀을 거치지 않고 힙 영역에 항상 새로운 인스턴스를 생성한다.

## 3. String의 불변성 (Immutable)
* `String`은 불변 객체이므로 한 번 생성된 이후에는 내부 값을 절대 변경할 수 없다.
* `concat()` 등 문자를 조작하는 메서드를 호출하면 기존 객체가 변하는 것이 아니라, 변경이 적용된 **새로운 String 객체를 생성하여 반환**한다.
* **불변으로 설계된 이유**: 문자열 풀에서 하나의 문자를 여러 변수가 공유할 때, 특정 변수에서 값을 변경하면 다른 변수들의 값도 의도치 않게 바뀌어버리는 **사이드 이펙트(Side Effect)를 방지**하기 위함이다.

## 4. StringBuilder (가변 객체)
* **String의 단점**: 문자를 자주 더하거나 변경하면 사용되지 않는 중간 객체가 계속 생성되고 버려져, CPU와 메모리(GC) 자원을 심하게 낭비한다.
* **해결책**: 내부에 변경 가능한 배열을 가지고 있어 값을 직접 수정할 수 있는 가변(Mutable) 객체인 `StringBuilder`를 사용한다.
* **최적화 시 주의점**:
    * 자바 컴파일러는 단순한 문자열 더하기(`+`) 연산을 내부적으로 자동 최적화해 준다.
    * 하지만 **반복문(Loop) 안에서 문자열을 계속 더할 때**는 자동 최적화가 이루어지지 않아 매번 새 객체가 생성된다. 이때는 반드시 `StringBuilder`를 명시적으로 사용해야 한다.

## 5. 메서드 체이닝 (Method Chaining)
* 메서드 실행 결과로 **자기 자신의 참조값(`this`)을 반환**하여, 반환값에 계속해서 점(`.`)을 찍고 다음 메서드를 이어서 호출하는 프로그래밍 기법이다.
* 코드를 매우 간결하고 읽기 쉽게 만들어 준다.
* `StringBuilder`의 주요 메서드(`append()`, `insert()`, `delete()`, `reverse()`)는 모두 메서드 체이닝을 지원한다.
    * *예시:* `sb.append("A").append("B").reverse();`

## 6. String 주요 메서드 요약
* **정보 조회**
    * `length()`: 문자열의 길이 반환
    * `isEmpty()`: 길이가 0인지 확인
    * `isBlank()`: 공백만 있거나 비었는지 확인 (자바 11+)
* **검색 및 비교**
    * `equals()`: 동등성 비교 (대소문자 무시는 `equalsIgnoreCase()`)
    * `contains()`: 특정 문자열 포함 여부 확인
    * `indexOf()`: 특정 문자열이 처음 등장하는 인덱스 반환
* **조작 및 변환**
    * `substring()`: 문자열을 지정한 위치부터 자르기
    * `replace()`: 특정 문자를 다른 문자로 대체
    * `trim()` / `strip()`: 양쪽 공백 제거 (`strip()`은 유니코드 공백까지 제거)
* **분할 및 조합**
    * `split()`: 정규 표현식을 기준으로 문자열을 배열로 분할
    * `join()`: 주어진 구분자를 사용하여 여러 문자열을 하나로 결합
