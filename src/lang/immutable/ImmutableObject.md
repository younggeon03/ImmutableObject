# 자바 중급 - 불변 객체 (Immutable Object)

## 1. 기본형 vs 참조형의 공유
* **기본형(Primitive Type)**: 하나의 값을 여러 변수에서 절대로 공유하지 않습니다. 값을 대입할 때 항상 '값 복사'가 일어나기 때문입니다.
* **참조형(Reference Type)**: 하나의 인스턴스를 참조값을 통해 여러 변수에서 공유할 수 있습니다.

## 2. 공유 참조와 사이드 이펙트 (Side Effect)
* **정의**: 주된 작업 외에 추가적인 부수 효과를 일으키는 것을 의미합니다.
* **문제점**: 여러 변수가 하나의 객체를 공유(공유 참조)하고 있을 때, 한쪽에서 객체의 값을 변경하면 의도치 않게 다른 변수가 참조하는 값도 바뀌어 버립니다.
* **해결책**: 처음부터 서로 다른 인스턴스를 참조하게 하거나, 객체를 **불변**으로 설계해야 합니다.

## 3. 불변 객체 (Immutable Object)
* **개념**: 객체의 상태(내부 필드 값)가 생성 이후 절대로 변하지 않는 객체입니다.
* **설계 방법**:
    1. 필드 변수를 `final`로 선언합니다.
    2. `setValue()`와 같은 수정자(Setter)를 제거합니다.
* **장점**: 사이드 이펙트를 원천 차단하여 코드의 안정성을 높이고 디버깅을 쉽게 만듭니다.

## 4. 불변 객체의 값 변경 방식
* 불변 객체는 기존 값을 변경할 수 없으므로, 변경이 필요한 경우 **기존 값에 변경사항을 더한 새로운 객체를 생성하여 반환**해야 합니다.
* **관례**: 새로운 객체를 반환하는 메서드는 보통 `withXxx()`라는 이름을 사용합니다. (예: `withYear(int year)`)

## 5. 정리
* 자바의 `String`, `Integer`, `LocalDate` 등이 대표적인 불변 객체입니다.
* 모든 클래스를 불변으로 만들 필요는 없지만, 캐시나 멀티 쓰레드 안정성이 필요한 경우 적극 활용됩니다.
